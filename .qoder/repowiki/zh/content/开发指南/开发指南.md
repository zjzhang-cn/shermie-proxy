## 简介
本开发指南面向希望扩展或定制 shermie-proxy 的开发者，目标是帮助你快速理解项目结构、核心模块职责与交互关系，并提供添加新协议处理器的完整流程、调试技巧、测试策略、代码质量要求、开发环境搭建与构建流程，以及贡献代码的最佳实践。

## 项目结构
项目采用按功能域分层的组织方式：
- Contract：契约与接口定义（当前仅定义通用处理器接口）
- Core：核心业务逻辑与协议处理（HTTP/HTTPS/WS/WSS/TCP/SOCKS5）
- Log：日志模块
- Utils：平台与工具函数
- Root：入口程序与文档

## 架构总览
Shermie-Proxy 采用“统一入口 + 协议识别 + 多处理器委派”的架构。入口负责参数解析与事件注册；服务器根据首包特征选择处理器；处理器内部通过事件回调实现可插拔的数据拦截与修改；证书与缓存模块支撑TLS与WS场景。

```mermaid

participant Handler as "具体处理器(HTTP/SOCKS5/TCP)"
participant Remote as "远端服务"
Client->>Server : "建立TCP连接"
Server->>Server : "读取首包并识别协议"
Server->>Handler : "委派处理"
alt HTTP/HTTPS/WS
Handler->>Remote : "建立目标连接/升级WS"
Handler-->>Client : "回写响应/WS消息"
else SOCKS5
Handler->>Remote : "握手并转发"
Handler-->>Client : "转发数据"
else TCP
Handler->>Remote : "直连目标并转发"
Handler-->>Client : "转发数据"
end
```

### 组件：ProxyServer（代理服务器）
- 职责
  - 监听端口、多路监听、连接接受与分发
  - 协议识别与处理器委派
  - 提供事件回调钩子，供上层拦截与修改
- 关键点
  - 事件类型定义集中在结构体字段中，便于统一管理
  - 协议识别通过首包前缀判断，避免复杂状态机
  - 证书安装/卸载与系统代理设置（Windows）

### 组件：ProxyHttp（HTTP/HTTPS/WS/WSS）
- 职责
  - 读取HTTP请求，支持CONNECT隧道、TLS与WS升级
  - 请求/响应拦截与重写，支持gzip解码
  - WS握手与双向转发，支持压缩扩展
- 关键点
  - 通过事件回调实现请求/响应拦截
  - WS升级使用内置Upgrader，支持子协议与压缩
  - DNS解析与本地网卡绑定

```mermaid
flowchart TD
Start(["进入HTTP处理器"]) --> Peek["读取首行判断方法"]
Peek --> |CONNECT| SSL["建立TLS会话并尝试WS升级"]
Peek --> |其他| Req["读取请求体"]
SSL --> Upgrade{"是否升级WS?"}
Upgrade --> |是| WS["WS升级并双向转发"]
Upgrade --> |否| Forward["转发HTTP请求"]
Req --> InterceptReq{"是否注册请求拦截?"}
InterceptReq --> |是| ResolveReq["调用resolve重写请求"]
InterceptReq --> |否| Forward
Forward --> Resp["读取响应体(含gzip解码)"]
Resp --> InterceptResp{"是否注册响应拦截?"}
InterceptResp --> |是| ResolveResp["调用resolve重写响应"]
InterceptResp --> |否| Write["回写响应"]
WS --> End(["结束"])
Write --> End
```

## 性能考虑
- 并发监听与处理
  - 服务器采用多协程监听与处理，提升吞吐
- Nagle算法
  - 可配置是否启用Nagle，平衡延迟与带宽
- 缓冲与I/O
  - 使用bufio Reader/Writer减少系统调用
- DNS缓存
  - 减少重复解析开销
- 证书缓存
  - 域名级并发去重，避免重复生成证书
